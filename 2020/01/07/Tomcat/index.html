jdk下载页面：https://www.oracle.com/technetwork/java/javase/downloads/index.html

JDK环境变量配置：
变量名：JAVA_HOME
变量值：C:\Program Files\Java\jdk-11.0.4 （即为JDK路径）

Path系统变量添加：
;%JAVA_HOME%\bin（前面有个分号）
（如果Tomcat出现打开闪退的情况，试试改成 C:\Program Files\Java\jdk-11.0.4\bin; 即不要用%替换，其他的也一样）
（或填另一种值：;%JAVA_HOME%\bin;%JAVA_HOME%\jre\bin）
在变量最末尾添加时，若之前末尾无分号，则变量前面需添加分号）

新建一个名为CLASSPATH的环境变量
变量名：CLASSPATH
变量值：.;%JAVA_HOME%\lib\dt.jar;%JAVA_HOME%\lib\tools.jar;
（前面有个圆点和一个分号，圆点代表当前路径，下图因为我配置jmeter,所以也是加配置在classpath里，无妨）

验证是否成功：Win+R ->cmd -> java -version 显示jdk版本
javac 显示帮助页面
即配置成功

Tomcat :
下载地址：https://tomcat.apache.org/download-90.cgi（注意选择 Binary Distributions 下的zip文件而不是Source Code Distributions下的，前者是安装包，后者是源码）
目录介绍：
bin：可以执行文件。
conf：tomcat服务器的配置文件
lib：tomcat启动后需要依赖的jar包
logs：tomcat工作之后的日志文件
webapps：是tomcat布暑工程的目录。
work：jsp文件在被翻译之后，保存在当前这个目录下，session对象被序列化之后保存的位置

环境变量：
新建 JAVA_HOME 系统变量 ：C:\Program Files\Java\jdk-11.0.4（jdk目录）
启动：
tomcat目录/bin/startup.bat(window启动文件) 找到startup.bat 双击运行
dos环境输入 net start tomcat
dos环境下，cd至bin目录 catalina run 出错时可以看到提示
关闭：
第一种：Ctrl+C键 关闭Tomcat服务器
第二种：点击Tomcat窗口的右上角关闭按钮 （暴力停止服务器）
第三种：找到tomcat目录/bin/shutdown.bat文件，双击执行关闭Tomcat。
在浏览器中输入 127.0.0.1:8080（端口默认8080） 若显示Tomcat页面即搭建成功

修改端口
找到tomcat目录/conf/server.xml ，修改port的值，将port端口的值修改为80

第一种：把项目文件拷贝到tomcat/webapps目录下
